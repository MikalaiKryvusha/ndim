# Референс-коллекция: скриншоты живого NDim 1.x

> **Зачем.** Владелец (чат, 2026-07-11): «нарезать себе скриншотов старого ндим, чтобы коллекция
> всегда была под рукой» — источник вдохновения и сверки «как сделано сейчас» при проектировании
> экранов 2.0.
>
> **⚠️ Сами скриншоты НЕ коммитятся** (`.gitignore`: `design/reference-1x/*`): репозиторий
> публичный, а залогиненные экраны (Связи, Пространство) показывают имена настоящих людей.
> В git живёт только этот README.

## Именование

- `public-*` — экраны без логина (онбординг, велком, вход): снимаются автоматически,
  скрипт — `capture-1x-public.mjs` в scratchpad агента (Playwright, viewport 430×932).
- `app-*` — залогиненные экраны из браузера владельца (CDP, порт 9222): Дом, Связи,
  Пространство, Измерения («Все» и «Мой NDim ID»), Меню и подэкраны.
- `public-force-*` — экраны, показанные принудительно (SPA держит все страницы в DOM);
  на `auth_page` поэтому видны сразу обе формы (создание + вход) — в живом приложении
  они переключаются ссылками внизу.

## Состав коллекции (снято 2026-07-11, 25 кадров)

- **Публичные (8):** первый экран, онбординг ob1–ob4, велком, вход (обе формы разом —
  артефакт форс-показа), сброс пароля.
- **Приложение (14):** Дом · Связи · Пространство · Измерения «Все» · Измерения «Мой NDim ID»
  (вкладки — CSS-табы на `input[name=segment_selector]`: переключать через `#radio_2_my`, клик
  по div-панели не работает) · поиск измерений · предложить измерение · Меню · Поделиться ·
  Поддержка · Пожертвование · О проекте · Об авторе · Управление аккаунтом.
- **Раскрытые состояния (3):** карточка измерения (клик по названию → Описание, Автор, Год,
  Теги) · карточка связи (клик по имени → профиль гостя + параметры его пространства и
  **нашего общего**: размерность, диаметр, расстояние в звёздах и % от диаметра) · история
  версий в «О проекте» (дерево `<details>`, раскрывается `d.open = true`).

## Как пополнять

1. Публичные: перезапустить скрипт из scratchpad (или аналогичный) — безопасно, без логина.
2. Залогиненные: владелец запускает отдельный Chrome
   `chrome.exe --remote-debugging-port=9222 --user-data-dir="%LOCALAPPDATA%\ndim-debug-chrome"`,
   логинится в 1.x один раз; агент подключается Playwright'ом `connectOverCDP` и ходит по
   экранам **только читая** (навигация и снимки; в живом приложении настоящие пользователи —
   ничего не нажимать, что меняет данные: ни звёзды, ни сохранения, ни удаления).
