# EXPERIENCE — накопленный опыт агента

> Растущий журнал уроков агента. **Внешняя память о том, *что работает, а что нет*** — чтобы свежая,
> лишённая контекста сессия (или автономный цикл) никогда не повторяла тупик. Сверяйся с ним ПЕРЕД
> задачей; дописывай ПОСЛЕ значимой попытки (успешной **или** провальной). Грепай, а не листай.
>
> **Теги живут прямо в записи** (централизованного списка нет) — так один греп находит опыт напрямую:
> `grep '#loop' EXPERIENCE.md` · `grep -i '#context\|#build' EXPERIENCE.md` · `grep '❌' -A4 EXPERIENCE.md`
> · `grep 'EXP-0007' EXPERIENCE.md`. Переиспользуй существующий тег, если он подходит (грепни, чтобы
> посмотреть, какие уже в ходу).
>
> **Формат записи (коротко и удобно для грепа).** Новые сверху. Каждая запись начинается со стабильного
> идентификатора, ISO-даты, маркера исхода (`✅` / `❌` / `❌→✅`) и встроенных `#тегов`:
>
> ```
> ### EXP-0001 · 2026-01-01 · ✅ · #тег #область
> **Контекст:** одна строка — что делалось.
> **Пробовал / сделал:** подход, кратко.
> **Результат:** ✅/❌ — что произошло.
> **Урок:** переиспользуемый вывод (ради него запись и существует).   → ссылка: bugs/NN · ideas/NN · plans/NN
> ```
>
> Скилл: `/experience` (зафиксировать урок · вспомнить релевантные уроки).

### EXP-0019 · 2026-07-11 · ✅ · #svg #css #design #logo

в строке замены (и через bash-экранирование, и через PowerShell), потому что копировал контекст.

### EXP-0014 · 2026-07-10 · ✅ · #firestore #rules #tests #mutation
**Контекст:** правила безопасности Firestore — единственный сторож между клиентом и чужими данными
(сервера-фильтра у нас нет).
**Пробовал / сделал:** написал тесты правил так, чтобы они проверяли **отказы**, а не разрешения, и
прогнал восемь мутаций правил: `pending` ≈ `accepted`; отправитель принимает свой запрос дружбы;
приватный бакет открывается группой с именем `private`; владелец пишет свои связи; чужая
подсказка `audience` читается всеми; чужие оценки по осям видны; оценка вне 0…10 проходит; снятие
умолчания `match /{document=**} { allow read, write: if false; }`.
**Результат:** ✅ — каждая мутация ловится. Последняя роняет 39 тестов из 56.
**Урок:** в правилах безопасности **сломанное разрешение видно сразу** (приложение перестаёт
работать), а **сломанный отказ не виден никогда** — до дня, когда кто-то прочитает чужие данные.
Поэтому тестов на «нельзя» должно быть больше, чем на «можно», и обязательна мутация: ослабь правило —
тест обязан покраснеть. Отдельно проверяй попытки **обойти** правило (группа с зарезервированным
именем `private`/`friends`), а не только прямой доступ.   → ссылка: tests/rules/rules.test.ts

### EXP-0013 · 2026-07-10 · ❌→✅ · #tests #mutation #migration
**Контекст:** тесты границ значений (длина имени, «о себе», диапазон оценки) в `schema.test.ts`.
**Пробовал / сделал:** написал их красиво и параметризованно: `assert.doesNotThrow(() =>
validate(text(NAME_PART_MAX)))`, `assert.throws(() => validate(text(NAME_PART_MAX + 1)))`.
**Результат:** ❌ — при мутационной проверке выяснилось, что тихое ужесточение самой константы
(`50 → 40`) **не роняет ни одного теста**: они читают её же. А ужесточение границы означает, что
миграция отвергнет данные, которые люди уже заполнили. Добавил отдельный тест, прибивающий значения
1.x гвоздями (`assert.equal(NAME_PART_MAX, 50)`), — после этого мутация ловится.
**Урок:** тест, параметризованный тем самым значением, которое он проверяет, слеп к изменению этого
значения. Если константа — это **контракт** (совместимость с боевыми данными, лимит из требований),
её надо зафиксировать литералом хотя бы в одном тесте. Мутационная проверка находит такие дыры, обычный
зелёный прогон — нет.   → ссылка: src/lib/model/schema.test.ts

### EXP-0011 · 2026-07-10 · ❌→✅ · #typescript #node
**Контекст:** класс ошибки с параметром-свойством: `constructor(readonly property: string)`.
**Пробовал / сделал:** обычный для TypeScript приём.
**Результат:** ❌ — тестовый файл не загрузился целиком, а `tsc` выдал `TS1294: This syntax is not
allowed when 'erasableSyntaxOnly' is enabled`. Node исполняет TypeScript **стиранием типов** и
параметр-свойства стереть не может: они порождают код, а не только типы. То же относится к `enum`,

**Урок:** без сборки (Node 24 + `--strip-types`) TypeScript ограничен **стираемым** синтаксисом. Флаг
`erasableSyntaxOnly` в `tsconfig.json` ловит это на `npm run typecheck` заранее — держать включённым.
Поле объявляется явно, присваивается в конструкторе.

умеют перебирать значения map, и это был обход ограничения, а не каприз автора.
**Урок:** перед проектированием модели данных **проверь ограничения платформы по документации**, а не

как костыль. Хорошо понятое ограничение экономит месяц. → ссылка: researches/04 · interviews/002

### EXP-0009 · 2026-07-10 · ❌→✅ · #tests #math #rounding
**Контекст:** переносил ядро похожести из 1.x в TypeScript и писал тест «числового паритета».
**Пробовал / сделал:** посчитал ожидаемые значения в уме: `proximity = 72`, а «наивная» проверка
`round(proximity% × commonality% / 100)` должна была дать те же 48.
**Результат:** ❌ — тест упал. Точный расчёт: `proximity = 72.5034 → 73`, а наивное произведение

Упало не ядро, а мои ожидания.
**Урок:** ожидаемые значения в тестах на арифметику **вычисляй, а не прикидывай** — одной строкой
`node -e`. И никогда не считай итоговую метрику из уже округлённых промежуточных: разница в 1 п.п.
ломает паритет с боевыми данными. Провалившийся тест здесь оказался ценнее зелёного: он превратился в
