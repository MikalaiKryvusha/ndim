# /check-backlog — ревизия беклога (bugs/ и plans/) и простановка DONE

Со временем в `bugs/` и `plans/` накапливаются файлы, которые уже сделаны, но не помечены `DONE`, — и
листинг файлов перестаёт показывать, что осталось. Этот скилл наводит порядок: собирает актуальный
открытый список И помечает по-настоящему закрытые файлы тегом `DONE`.

Опирается на конвенцию DONE-тега в имени файла (см. `AGENT_GUIDE.md` → «Беклог и DONE-тег»).

## Что делать

1. **Собери все файлы беклога:**
   - `ls bugs/` — все документы багов.
   - `ls plans/` и `ls ideas/` — идеи, фичи, рабочие журналы и планы.
   - ⚠️ В корне и в `plans/` есть СПРАВОЧНЫЕ документы (не задачи): `MASTER_PLAN.md`, `GOAL.md`,
     обе карты, заметки `researches/`. Их DONE **не помечать** — это живые справочники, а не
     закрываемые задачи. Помечай только конкретные баги, идеи, фичи и задачи.

2. **Раздели на два списка по имени файла:**
   - **Открытые (без `DONE` в имени)** — кандидаты в беклог «к реализации». Отметь их.
   - **Помеченные `DONE`** — уже закрыты, пропусти.

3. **Для КАЖДОГО открытого файла определи реальный статус** (не угадывай!):
   - Прочитай документ: есть ли внутри статус ✅/ЗАКРЫТО/РЕАЛИЗОВАНО/ИСПРАВЛЕНО/ДОКАЗАНО?
   - Сверься со `STATUS.md` и, если нужно, с кодом и `git log` — действительно ли это сделано?
   - **Если действительно СДЕЛАНО и подтверждено** → проставь тег DONE (шаг 4).
   - **Если НЕ сделано / частично / только исследование (🔧/🟡/🔬)** → оставь открытым, добавь в
     текущий беклог (с короткой пометкой «что осталось»).
   - **Если непонятно** → НЕ помечай; оставь открытым с пометкой «нужна проверка».

4. **Простановка DONE (только для подтверждённо закрытых):**
   - Переименуй, вставив `DONE` после номера и сохранив историю:
     `git mv bugs/13_detach_crash.md bugs/13_DONE_detach_crash.md` (номер не меняй; формат
     `<NN>_DONE_<имя>.md`).
   - **Допиши секцию статуса внутрь документа**, например:
     ```
     ## ✅ СТАТУС: DONE (<дата>)
     Что сделано: <краткая сводка фикса или реализации>.
     Как проверено: <сборка / стенд / измерение / хеш коммита / итерация цикла>.
     ```
   - Закоммить (в цикле — по обычной коммит-дисциплине; иначе — пачкой в конце ревизии).

5. **Выдай сводку:**
   - Короткий список ОТКРЫТЫХ задач (беклог) с приоритетом: доделать начатое > баги > новые идеи.
   - Список файлов, помеченных `DONE` в эту ревизию.
   - Всё, что осталось с пометкой «нужна проверка» (неясный статус).

## Заметки
- Ревизия аккуратная: помечай DONE только то, что РЕАЛЬНО закрыто и проверено — лучше оставить
  открытым, чем ошибочно закрыть.
- Справочные документы никогда не помечаются DONE.
- В автоциклах и при `/refresh-context` вызывай раз в несколько итераций, а не каждую.
- Если всплыли вопросы уровня человека — заведи в `interviews/` + отметь `STATUS.md`, не решай вслепую.
