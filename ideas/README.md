# `ideas/` — предложения фич и улучшений

Детальные идеи о том, *что* строить — обычно узкий срез проекта, описанный достаточно хорошо, чтобы
агент мог по нему реализовать. Чаще всего автор — **человек**, но агент тоже предлагает идеи. Один
`NN_<имя>.md` на идею.

**Для человека (владельца):** это твоя главная авторская директория. Положи сюда идею с описанием
того, чего ты хочешь; агент причешет её в чистый структурированный вид и реализует по ней. Идея —
это часть продуктового **видения**, поэтому агент реализует её только после твоего одобрения.

**Для ИИ-агента:** читай идеи владельца, исправляй опечатки, минимально переструктурируй для ясности,
затем реализуй. Когда идея появляется у *тебя* — заведи её здесь со статусом «❓ ожидает одобрения
владельца» (скилл: `/propose-idea`) и **не** реализуй до одобрения. После реализации впиши в файл
статус и дату и пометь тегом `DONE` (`git mv NN_x.md NN_DONE_x.md`).

> На NDim Space действует особая проверка: идея, которая повышает вовлечённость или монетизацию ценой
> качества поиска людей, противоречит `GOAL.md`. Такую идею не предлагают — её переформулируют.
