# NDim Space

Этот проект обёрнут фреймворком **KAIF**. Рабочий язык — **русский**.

## Перед каждой задачей

1. Прочитай **`AGENT_GUIDE.md`** — это канон проекта: правила, карта, команды, конвенции.
2. Прочитай **`STATUS.md`** — текущее состояние и чеклист «где продолжить».
3. Думай по **`PHILOSOPHY.md`** (ПРОСТОТА: KISS + бритва Оккама). Застрял → перепойми задачу, а не
   наращивай сложность.
4. Чини дефекты по **`BUG_FIXING_FRAMEWORK.md`** (правило трёх попыток → `/bug-research`).
5. Вспомни уроки: грепни **`EXPERIENCE.md`** по тегам задачи.

Не дублируй канон здесь — читай его в `AGENT_GUIDE.md`.

## Четыре вещи, которые ломают этот проект чаще всего

- **Старого кода в проекте нет.** NDim Space 1.x целиком живёт в приватном
  `MikalaiKryvusha/ndim-old`. Здесь только 2.0. Нужно посмотреть, как было, — иди туда
  (`gh repo clone MikalaiKryvusha/ndim-old`). Ключевое знание уже выжато в
  `researches/02` (модель данных) и `researches/03` (математическое ядро).
- **Цель — добро, а не выручка.** `GOAL.md` прямо отвергает механики удержания и подписки. Решение,
  повышающее вовлечённость ценой качества поиска людей, противоречит проекту.
- **UI/UX не решается агентом.** Перед реализацией любого брендового или UX-решения агент рисует
  **4 варианта макета**, показывает автору, тот утверждает один (возможно с правками). Сам агент
  делает только самое очевидное. См. `AGENT_GUIDE.md` → «Дизайн».
- **PowerShell 5.1.** Нет `&&` и `||` (используй `;` и `if ($?) { }`). Скрипты `.ps1` с кириллицей
  обязаны быть в UTF-8 **с BOM**. Пути с пробелами git экранирует — сверяй с `-c core.quotepath=false`.

## Перед каждым push

`npm run kaif:check` — падает, если в git попал секрет или ПДн. Здесь это уже случалось однажды.
