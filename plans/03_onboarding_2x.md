## Целевой путь пользователя (сквозной)

```
Лендинг ──▶ Демо похожести (без аккаунта, 30 секунд, ядро в браузере)
              │  «Создай своё пространство»
              ▼
        Гостевой режим (anonymous auth: свои оси, оценки, своё пространство;
              │   никому не виден; труд копится в Firestore под гостевым UID)
              │  логичный момент: «сохранить навсегда» / «стать видимым» / «дружба»
              ▼
        Аккаунт (Google / волшебная ссылка / OAuth-сервисы; linkWithCredential:
                 UID тот же, ВЕСЬ труд гостя сохраняется)
```

Каждый этап воронки пишет счётчик (В4) — увидим, где рвётся.

### Этап 2 — Гостевой режим (`ideas/11`, часть 1) 🔧 в работе (низ готов, 2026-07-11)

- 🔲 Firebase Anonymous Auth во всех окружениях: стенд (эмулятор Auth уже на :9099) → прод.
- ✅ **Правила**: `isGuest()` по `sign_in_provider == 'anonymous'` + честный флаг `guest` на
  точке (`honestGuestFlag`: гость не скроет, полноценный не прикинется). Разрешено: свои
  `points/`+оценки, каталог осей, свои relations, приватные бакеты. Запрещено: публикация
  бакетов everyone/friends, группы, `audience`, дружба, заявки на оси (В3 = A). 17 тестов
  отказов (73 всего), 3 мутации правил пойманы.
- ✅ **Вычислитель**: точки с `guest: true` не попадают ни в чей топ (даже другого гостя),
  но сам гость получает свой топ против публичных точек. Тест `npm run test:calc` (5 проверок
  на эмуляторе), мутация «фильтр снят» ловится. Точка входа теперь срабатывает только при
  прямом запуске — модуль импортируем тестами.
- ✅ **Автоочистка сирот**: `cleanupStaleGuests()` в начале каждого цикла вычислителя —
  guest-точки с `lastSync` старше 30 дней (и не dirty) удаляются целиком (точка, оценки,
  relations, users-дерево). Труд полноценных людей неприкосновенен при любом возрасте —
  это охраняется тестом (мутация «фильтр гостей снят» ловится). `test:calc` — 9 проверок.
- ✅ **UI-флоу гостя утверждён** (2026-07-11, `design/guest-flow-mockups.html`): база —
  **V1 «Тихий бейдж»** с правками владельца (карточка центрирована как в V4, главный текст
  «Пока что ты гость, твои результаты не сохранены», кнопка **[Сохранить результаты]**).
  Метафора: гость = пунктирный контур, аккаунт = сплошной. Правки уже внесены в макет.
- 🔲 Реализация UI гостя по утверждённому V1: пилюля «◌ гость» в шапке → центрированная
  честная карточка; мост из демо («Создай своё пространство» → мгновенный анонимный вход).

### Этап 3 — Аккаунт без пароля (`ideas/11`, часть 2) 🔲

- Апгрейд гостя: `linkWithCredential`/`linkWithPopup` — UID сохраняется. Строго link, не signIn.
- Методы (В2): **Google** (один тап) + **email-ссылка** (passwordless). Другие известные
  сервисы — по мере: Apple требует платного Apple Developer, Facebook — заведения приложения;
  добавляем, когда цена оправдана. Паролей в продукте нет вообще.
- Экран/модал входа — 4 макета на утверждение (отдельный набор от демо-блока).
- Правила: полный доступ открывается только не-анониму. Тесты отказов и «апгрейднутый гость
  сохранил доступ к своим данным».
- e2e на стенде: гость заполняет оси → апгрейд (эмулятор Auth умеет и Google-заглушку,
  и email-link) → данные на месте, UID тот же.

## Критерий готовности плана

Человек с рекламы проходит путь «лендинг → потрогал → гость → аккаунт» без единого пароля и
без потери введённого, каждый переход измерим, ни одно правило приватности не ослаблено
(мутационные тесты правил зелёные).
