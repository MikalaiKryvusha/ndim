# `plans/` — детальные пошаговые планы

Детальные планы отдельных кусков работы: конкретных шагов мастер-плана, фич, идей, багов,
исследований, процедур. **`MASTER_PLAN.md`** (корень проекта) — высокоуровневая дорожная карта;
`plans/` хранит приближённые планы, которые реализуют её шаги. Один `NN_<имя>.md` на план.

**Для человека (владельца):** писать сюда не обязательно — планы обычно дело агента. Можешь положить
план, если хочешь порулить тем, *как* что-то будет сделано. Читай их, чтобы увидеть намеченный
агентом подход до того, как он его выполнит.

**Для ИИ-агента:** перед нетривиальной работой напиши сюда короткий план и следуй ему. Нумеруй файлы
(`NN_<имя>.md`). Законченный и проверенный план получает тег `DONE` в имени файла
(`git mv NN_x.md NN_DONE_x.md`) плюс секцию статуса. Справочный материал (не закрываемая задача)
тегом `DONE` не помечается.
