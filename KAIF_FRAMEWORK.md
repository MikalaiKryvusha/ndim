## Что такое KAIF

KAIF (Krinik AI Framework) — **устойчивый к потере контекста, дисциплинирующий автономию операционный
фреймворк для тандема «человек — ИИ»**. Он выносит рабочую память и дисциплину агента наружу, в этот
репозиторий: небольшой набор markdown-документов, конвенции директорий и повторяемые слэш-скиллы. За
счёт этого любая свежая сессия агента возобновляет работу с полным контекстом, действует автономно в
понятных границах и накапливает знание вместо того, чтобы его терять. Это не код; это *процесс,
зафиксированный файлами, которые агент читает*.

## Зачем он здесь — что он даёт этому проекту

- **Никаких холодных стартов.** Новая сессия читает `AGENT_GUIDE.md` + `STATUS.md` и сразу продуктивна.
- **Знание переживает сессию.** Баги, решения, исследования и идеи становятся долговечными документами,
  а не потерянным чатом.
- **Ограниченная автономия.** Агент сам мелет беклог и эскалирует только решения уровня владельца.
- **Общий метод.** Человек — визионер (`GOAL.md`), агент — исполнитель; KAIF — интерфейс между ними.

Для NDim Space это особенно важно: проект переживает переход от версии 1.x, написанной вручную
энтузиастом, к NDim Space 2.0. Такой переход — марафон на много сессий, и без внешней памяти он бы
рассыпался. `legacy/` хранит идею, KAIF хранит путь к её новому воплощению.

## Как это работает здесь — движущие части

| Часть | Роль в этом проекте |
|-------|---------------------|
| `AGENT_GUIDE.md` | Канон, который агент читает перед каждой задачей. Здесь же — инвариант «`legacy/` только для чтения». |
| `PHILOSOPHY.md` | Как агент мыслит (KISS + Оккам + набор принципов). Раздел «Карма» здесь — продуктовое требование, а не метафора. |
| `BUG_FIXING_FRAMEWORK.md` | Как агент чинит дефекты (правило трёх попыток → `/bug-research`). |
| `TESTING_FRAMEWORK.md` | Канон тестирования: семь принципов, маркеры `[NOT-TESTED]`/`[TESTED: …]` (агентский документ, англ.). |
| `GOAL.md` / `MASTER_PLAN.md` | Видение владельца и фазовый путь к нему. |
| `STATUS.md` | Живое состояние — обновляется после каждой значимой задачи. |
| `PROJECT_STRUCTURE_EXTERNAL_MAP.md` / `PROJECT_ARCHITECTURE_INTERNAL_MAP.md` | Внешняя и внутренняя карты. Во внутренней разобрано математическое ядро. |
| `EXPERIENCE.md` | Журнал уроков агента, грепается по тегам. |
| `plans/ ideas/ bugs/ researches/ interviews/ homeworks/` | Директории знаний (у каждой свой README). |
| `.claude/skills/` | Повторяемые ритуалы (`/resume`, `/pause`, циклы, …) — 26 скиллов, включая семейство дисциплины исполнения `fable-*`. |
| `.kaif/kaif.json` | Маркер развёртывания: версия, сфера, агенты, отслеживание. |
| `.kaif/spheres/` | Библиотека сфер (что значит «проверить наблюдением» в каждой); проект живёт в `programming`. |
| `tools/` | Скрипты развёртывания и сопровождения: скачивание ядра, перепроверка, npm-хендлы. |

## Скрипты развёртывания (остаются в проекте)

| Скрипт | Что делает |
|--------|-----------|
| `tools/02-kaif-fetch.mjs` | Скачивание ядра `KAIF.md` с origin и сверка по версии и sha256 |
| `tools/03-kaif-verify.mjs` | Перепроверка развёртывания: плейсхолдеры, локализация, канонические `name:` скиллов, маркер, **утечка секретов в git** |
| `tools/kaif.mjs` | Бэкенд npm-хендлов `kaif:*` |

> Скрипт переноса наследия (`01-copy-legacy.ps1`) удалён: он свою работу сделал, а старого кода в
> проекте больше нет (`ideas/01`). Процедура задокументирована в `researches/01`.

## Жизнь с KAIF (жизненный цикл)

`/kaif-version` (проверить обновления) · `/kaif-update` (бережная миграция с origin) · `/kaif-fork`
(развивать свой) · `/kaif-switch-origin` · `/kaif-remove` (частичное сохраняет твои артефакты, или
полное — всегда бережно). Подкреплено хендлами `npm run kaif:*`.

Ежедневное вождение: `/resume` (начать) · `/pause` (свернуться) · `/autoloop` · `/dayloop` ·
`/nightloop` (автономная работа) · `/report-bug` · `/propose-idea` · `/interview` · `/revision` ·
`/fix-vision` · `/what-next` · `/help-kaif` · `/release`. Дисциплина исполнения: `/fable-method` ·
`/fable-loop` · `/fable-judge` · `/fable-domain`.

Не знаешь, с чего начать — набери `/help-kaif`.
