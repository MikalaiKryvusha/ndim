# NDim Space — Внешняя карта структуры

> **ВНЕШНЯЯ карта: как проект выглядит снаружи** — его директории, файлы, перекрёстные ссылки и
> зависимости между ними. Это карта «где что лежит», которую читает свежая сессия, чтобы
> сориентироваться. Её напарник — `PROJECT_ARCHITECTURE_INTERNAL_MAP.md` (*внутренняя* логическая
> архитектура: абстракции и как они взаимодействуют).
>
> Сфера проекта — **программирование** (веб-приложение). Держи карту синхронной с реальным деревом.
> **Живой справочник — никогда не помечается DONE.**

**Последнее обновление:** 2026-07-10

## Где старый проект

Версии 1.x в этом репозитории **нет**. Она целиком в приватном `MikalaiKryvusha/ndim-old` — по решению
владельца новый проект начинается с чистого листа.

```bash
gh repo clone MikalaiKryvusha/ndim-old        # приватный архив (включая NDim_Space/ — файлы гравировки)
git clone .private/ndim-1.x-history.bundle    # то же локально, 825 коммитов
```

Ходить туда обычно не нужно: ключевое знание выжато в `researches/` (см. таблицу ниже).

## Перекрёстные ссылки и правила зависимостей

- **Старых файлов в проекте нет.** Всё, что относится к 1.x, живёт в приватном `ndim-old` и в
  `researches/`. Не тащи оттуда файлы обратно — тащи знание.
- **`.private/` никогда не попадает в git.** Проверяется автоматически: `npm run kaif:check` падает,
  если файл с секретом или ПДн оказался под контролем версий.
- **Документы ссылаются вниз, а не вверх.** `GOAL.md` → `MASTER_PLAN.md` → `plans/NN_*.md`. Обратный
  поток запрещён: план не переопределяет цель.
- **Одно каноническое написание.** Имена, пути, URL — как в `AGENT_GUIDE.md` → «Идентичность проекта».
- **Знание живёт в файлах, а не в чате.** Дефект → `bugs/`. Решение владельца → `interviews/`.
  Урок уровня подхода → `EXPERIENCE.md`. Глубокое исследование → `researches/`.
- **DONE-тег.** Закрытые файлы в `bugs/`, `ideas/`, `plans/`, `homeworks/` переименовываются через
  `git mv NN_x.md NN_DONE_x.md`. `GOAL.md`, `MASTER_PLAN.md`, карты и `researches/` — живые
  справочники, их DONE не помечают никогда.
- **Верхний уровень — только ключевое.** Ни временных файлов, ни старого кода, ни черновиков.

## Точки входа

Куда смотреть новичку (человеку или агенту), по порядку:

1. **`GOAL.md`** — зачем всё это существует. Без него остальное бессмысленно.
2. **`STATUS.md`** — где мы сейчас и что делать следующим шагом.
3. **`AGENT_GUIDE.md`** — по каким правилам здесь работают.
4. **`PROJECT_ARCHITECTURE_INTERNAL_MAP.md`** — как система мыслит: измерения, пространства, похожесть.
5. **`researches/03_similarity_core_1x_source.md`** — сама идея, в 100 строках кода.
6. **`researches/02_firestore_data_model_1x.md`** — с какими данными мы обязаны считаться.
