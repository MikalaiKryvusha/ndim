# Bug 12 — Мобильная оболочка: хедер/футер не во всю ширину, нижняя панель уезжала вниз

**Статус:** ✅ DONE (2026-07-16, в бою) · **Источник:** волна 08 (п. 3), ручной тест владельца

## Симптом
(а) Шапка и нижняя панель не растянуты на всю ширину экрана. (б) Нижняя навигация
выталкивалась контентом за нижний край — на длинных экранах её не видно без скролла.

## Причина
На всех экранах оболочка `.screen { max-width: 430px; margin: 0 auto }` зажимала и шапку,
и панель. `BottomNav` стоял в потоке ПОСЛЕ контента без позиционирования.

## Фикс (сделан)
`BottomNav.svelte`: `position: sticky; bottom: 0` + `z-index` + safe-area (sticky сам
резервирует место в потоке — без спейсеров). Оболочки шести экранов (`profile`, `relations`,
`dims`, `space`, `menu`, `DocShell`): `max-width` снят со `.screen`, контент зажат колонной
458px по центру (`.body`). Десктоп (≥1024px, канон V2) не тронут.

## Проверка (пройдена)
`node tools/verify-bugs08.mjs`: rect'ы шапки/панели = ширине вьюпорта (390 и 800), панель
в кадре до и после скролла, контент центрирован. Смоук прода зелёный.

## Хвосты, вскрытые фиксом
- [[20_gap_below_bottom_nav]] — на коротком контенте появляется пустота (волна 09, п. 2);
- [[22_bottomnav_translucent]] — прозрачность панели стала заметна: контент скроллится ПОД ней.
