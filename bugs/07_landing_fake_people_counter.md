# Bug 07 — Лендинг врёт: «С нами уже 2 184 человека» зашито в код

**Статус:** 🔧 фикс сделан (литерал изъят, витрина живая), хвост — тесты и боевой контейнер по `plans/05` ·
**Заведён:** 2026-07-12 (агентом, при аудите живого прода)
**Важность:** высокая — прямое враньё на лице продукта. Противоречит `GOAL.md` (честность) сильнее
любого визуального дефекта.

## Симптом

Живой https://ndimspace.app говорит: «С нами уже **2 184 человека** — и каждый день приходят
новые». Число зашито литералом (`src/routes/+page.svelte:100`:
`joinedCount: { ru: '2 184 человека', en: '2,184 people' }`). В боевой базе после миграции —
**331 человек** (`STATUS.md`, сверка cutover 2026-07-12). Число выдумано и не меняется.

## Криминалистика

- `src/routes/+page.svelte:12` — `TODO(фаза 5): счётчик людей — живая метрика из
  space/public_metrics`. Фаза 5 (боевой запуск) состоялась 2026-07-12 — TODO не всплыл.
- В 1.x эта метрика была **живой**: текст владельца «Присоединяйтесь! С нами уже `N` человек…»
  подставлял число из документа `space/public_metrics`, читаемого **без авторизации**
  (`researches/05_onboarding_texts_1x.md:26,78`). То есть паритет с 1.x здесь тоже нарушен.
- Модель 2.0 такого документа не пишет: `space/stats` есть, но читается только вошедшими
  (правила), лендинг же открывается анонимом и Firebase SDK в свой бандл не тянет
  (канон воронки — динамический импорт).

## Корневая причина

Заглушка «на потом» пережила «потом». Лендинг делался до боевых данных, число вписали литералом
с TODO — а боевой запуск прошёл без ревизии TODO-меток на лице продукта. Выдуманное число никакой
тест не ловил: e2e проверяли наличие текста, а не его правдивость.

## План фикса

1. Сервер синхронизации пишет `space/public_metrics` (одно поле `people` — число полноценных
   людей, гости не в счёт — как в счётчиках `space/stats`) рядом с остальной статистикой.
2. Правила: `space/public_metrics` читается **без авторизации** (паритет с 1.x — в агрегате нет
   ничего персонального), запись клиенту запрещена. Тесты: чтение анонимом ✓, запись клиентом ✗.
3. Лендинг после отрисовки читает документ динамическим импортом (как воронка — SDK в бандл
   не попадает) и подставляет живое число; пока числа нет (или Firebase недоступен) — строка
   не показывает выдуманного значения (честная деградация: без числа).
4. e2e: в пререндеренном HTML лендинга нет литерала «2 184»; мутация «вернуть литерал» роняет тест.
