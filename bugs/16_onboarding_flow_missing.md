# Bug 16 — Онбординг-флоу («ОБ флоу») внутри продукта так и не реализован

**Статус:** 🔴 OPEN (пачка 4 — сначала макеты владельцу) · **Источник:** волна 08 (п. 7)

## Симптом
Пошагового знакомства новичка с продуктом нет. В 1.x был онбординг-флоу — экраны
`ob1_page`…`ob4_page` + `welcome_page`: добро пожаловать → NDim ID → Настоящие связи →
Руководство. В 2.0 эти тексты ушли на лендинг, а сам внутренний флоу не построен
(plans/03 закрыл воронку демо → гость → аккаунт, но не знакомство ПОСЛЕ входа).

## Связь с волной 09
Владелец уточнил рамку (bugs/09, п. 1): экран «Войдите в Пространство» должен быть
ОТДЕЛЬНЫМ экраном В РАМКАХ ОБ-флоу, в стиле лендинга — см. [[19_signin_not_own_screen]].
Проектировать их надо вместе: вход — одна из станций онбординга.

## План фикса
1. Разведка: снять флоу 1.x целиком из `ndim-old` (экраны, переходы, когда показывается,
   как пропускается) — дописать в `researches/05`. Тексты уже сняты дословно.
2. 4 макета (карусель как в 1.x / чек-лист первых шагов / подсказки по месту / гибрид) —
   вместе с экраном входа (bugs/19). Утверждение владельцем.
3. Реализация по утверждённому; онбординг не должен вернуть трение, из-за которого
   флоу 1.x бросали (researches/10, ideas/09).

## План проверки
Playwright-сценарий нового человека: гость → онбординг показан один раз → пропускается →
не показывается старым. Скриншоты каждой станции.
