# Bug 42 — Виджет «Сервер синхронизации» смешивает сердцебиение с полной синхронизацией

**Статус:** 🔴 открыт · **Заведён:** 2026-07-24 (скриншот владельца: «я не понимаю, почему,
но вот что происходит») · **Важность:** высокая — виджет систематически ВРЁТ владельцу о
поведении сервера; ровно эта ложь питала и исходную жалобу идеи 14.

## Симптом (скриншот владельца, 2026-07-24 ~11:38 местного)

Блок «Полная синхронизация» показывает: «Последняя успешная — 11:38 · Выполнена за 4,4
секунды · Пользователей синхронизировано — 74 · Связей рассчитано — 2 340 ·
Запланированная — 11:39». Читается как «каждую минуту происходит полная синхронизация 74
человек» — при спокойном Пространстве. Владелец не понимает, почему.

## Криминалистика: каждая строка виджета честна по отдельности — вместе они лгут

Скриншот снят через ~2 минуты после перевыката вычислителя v0.2.0 (перезапуск 11:36:46
местного, полный проход с канонизацией тай-брейка — 74 топа записаны за 4,4 с). Дальше:

| Строка виджета | Откуда (поле `space/server`) | Что это НА САМОМ ДЕЛЕ |
|---|---|---|
| «Последняя успешная: 11:38» | `lastSuccessAt` | обновляется **каждым холостым циклом** («холостой цикл — тоже успех», bugs/33) — в 11:38 связи НЕ считались |
| «Выполнена за 4,4 с» | `durationMs` | длительность **полного прохода в 11:36** (перезапуск) |
| «Пользователей синхронизировано: 74» | `usersSynced` | топов записано полным проходом 11:36 (канонизация ничьих) |
| «Связей рассчитано: 2 340» | `relationsComputed` | от того же прохода 11:36 |
| «Запланированная: 11:39» | `nextRunAt = lastRunAt + intervalSeconds` | следующий **ЦИКЛ** (сердцебиение), а не следующая **полная** (она завтра ~11:36) |

Композиция: заголовок «Полная синхронизация» приклеивает минутное сердцебиение-успех
(строка 1, 5) к числам суточного события (строки 2–4). `src/routes/space/+page.svelte:141`
(заголовок блока), `src/lib/model/stats.ts:105` (докстрока `lastSuccessAt` до сих пор
обещает «цикл, в котором действительно считались связи» — после bugs/33 это неправда).

Завтра станет хуже: суточный полный проход запишет ~0 топов (diff-экономия, ideas/14) —
виджет скажет «Пользователей синхронизировано: 0», что владелец прочтёт как поломку,
хотя это триумф экономии («проверено 93, менять нечего»).

## Корневая причина

В 1.x админ-консоль показывала **раздельно** `sync_server_status` / `full_sync_status` /
`partial_sync_status` с отдельными метриками `last_full_sync_*` и `last_partial_sync_*`
(researches/13 §4, §6; `ndim_old/app.js:6204`). При переносе в 2.0 три блока сплющены в
ОДИН набор полей `space/server`, а экран честно отрисовал кашу. Дефект был малозаметен,
пока прод крутился с часовым интервалом; минутный цикл (выкат 2026-07-24) сделал его
кричащим.

## План фикса (по канону 1.x — раздельные события)

1. **Вычислитель** (`calculator/index.mjs`, `reportServer`): писать раздельно
   - `fullSync: { at, durationMs, pointsChecked, topsWritten, relationsComputed }` — только полным проходом;
   - `partialSync: { at, durationMs, dirtyProcessed, topsWritten }` — только обычным циклом с работой;
   - сердцебиение как есть: `lastRunAt`, `lastSuccessAt` (bugs/33 не трогаем), `intervalSeconds`.
   Легаси-поля (`usersSynced`, `durationMs`, `relationsComputed`) продолжают писаться, пока
   у людей в браузерах живут старые бандлы экрана.
2. **Модель** (`src/lib/model/stats.ts`): расширить `SyncServerDoc`, поправить лживую
   докстроку `lastSuccessAt`, добавить `nextFullSyncAt` (= fullSync.at + период полного прохода).
3. **Экран** (`src/routes/space/+page.svelte`): три честных блока в структуре 1.x —
   «Статус» (состояние · последний запуск · следующий цикл) · «Полная синхронизация»
   (последняя · выполнена за · точек проверено · топов записано · связей рассчитано ·
   следующая — ЗАВТРА) · «Обычные циклы» (последний успешный · изменившихся обработано ·
   топов записано). Вид и стиль карточки не меняются — только раскладка строк по блокам
   (косметика без новых сущностей; если владелец захочет иначе — макеты).
4. **Тесты**: test:calc — раздельные блоки пишутся своими событиями и не затирают друг
   друга; юнит stats — `nextFullSyncAt`.
5. **Проверка**: живой QA-прогон экрана «Пространство» (обе темы, две ширины, скриншоты),
   сверка чисел виджета с логом контейнера; после суточного прохода — «топов записано 0»
   читается как норма рядом с «точек проверено 93».
