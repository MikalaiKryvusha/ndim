# Bug 32 — Исчезла авторизация по паролю; поддержать все три способа входа сразу

**Статус:** 🔴 OPEN (пачка 4 — вместе с экраном входа) · **Источник:** волна 09 (п. 15)

## Симптом (слово владельца)
«Исчезла авторизация по паролю, которая была изначально. Если можно поддерживать
одновременно три типа авторизации: ссылка на почту, пароль, Google OAuth — реализуем все
три одновременно».

## Как сейчас
2.0 сознательно строился «без паролей» (Google + ссылка на почту + гость). Но у 331
человека из 1.x пароли СУЩЕСТВУЮТ (провайдер password в боевом Auth включён) — их дверь
отняли. Технически Firebase поддерживает все три способа одновременно — конфликтов нет.

## План фикса
1. Экран входа ([[19_signin_not_own_screen]]): добавить вход почта+пароль; для новых —
   создание пароля опционально (ссылки и Google остаются основными).
2. `data/account.ts`: `signInWithEmailAndPassword` (+ обработка ошибок человеческим языком),
   для гостя — `linkWithCredential(EmailAuthProvider.credential(email, password))` — UID и
   труд сохраняются (тот же канон link*, не signIn*).
3. Восстановление пароля — стандартное письмо Firebase (Action URL уже починен, EXP-0042).
4. Проектируется в одних макетах с экраном входа и ОБ-флоу (bugs/16, 19) — не плодить
   промежуточный экран, который тут же перерисуем.

## План проверки
Стенд, Playwright: вход паролем существующего (эмулятор Auth), неверный пароль — честная
ошибка, сброс пароля через oob-код эмулятора (EXP-0045), гость+пароль — UID сохранён.
