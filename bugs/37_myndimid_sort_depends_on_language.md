# Bug 37 — «Мой NDim ID»: смена языка меняет порядок карточек

**Статус:** 🔴 открыт · **Источник:** владелец, чат 2026-07-17

## Симптом (слово владельца)
«Когда на вкладке Мой НДИМ ИД переключаешь язык приложения — прыгают карточки, то есть язык
как-то влияет на сортировку — неправильно. Язык не должен влиять на сортировку. Измерения в
моём НДИМ ИД отсортированы строго по оценке — по убыванию вниз».

## Что не так сейчас
Сортировка «Мой NDim ID» — по убыванию своей оценки, а при РАВНЫХ оценках — по имени
измерения. Имя берётся на текущем языке (ru/en) → при смене языка равные по оценке карточки
пересортировываются и «прыгают». Канон владельца: порядок от языка зависеть не должен.

⚠️ Юнит-тест «"Мой NDim ID" — по убыванию СВОЕЙ оценки, а при равных — по имени»
(`src/lib/data/dims*.test.ts`) охраняет именно текущее (неправильное) поведение — фикс
обязан поменять и тест, иначе тот честно упадёт на правильном коде.

## План фикса
1. Найти компаратор ленты «Мой NDim ID» (`src/lib/data/dims.ts`) и место, где тай-брейк
   читает локализованное имя.
2. Сделать тай-брейк независимым от языка: при равных оценках — стабильный ключ,
   одинаковый на обоих языках. Кандидат — id измерения (детерминирован и неизменен);
   сверить с поведением живого 1.x (разведка по plans/06: как сортировал 1.x —
   `ndim-old`, `researches/12`).
3. Обновить юнит-тест: «порядок не меняется при смене языка» (прогнать компаратор на
   ru и en — последовательности идентичны) + мутация «вернуть тай-брейк по имени» падает.

## План проверки
Юниты (обновлённые) + стенд, живой Chrome: `/profile` → вкладка «Мой NDim ID», переключить
RU↔EN несколько раз — порядок карточек не шевелится (сравнение списков id до/после);
обе темы, две ширины, скриншоты.
