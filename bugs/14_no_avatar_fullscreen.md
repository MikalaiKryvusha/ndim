# Bug 14 — Фотографии не открываются в полноэкранный просмотр

**Статус:** 🔧 починен, проверен QA-прогоном на стенде (2026-07-16), ждёт выката ·
**Источник:** волна 08 (п. 5), ручной тест владельца

**Фикс:** лайтбокс в `Avatar.svelte` (тап → фото во весь экран, тап/Esc закрывают); в
«Связях» фото и имя — раздельные кнопки. **Стенд научился фотографиям**: эмулятор Storage
(порт 9199, `storage.rules` — черновик, в бой не выпущен), сид кладёт лица dev-пользователю
и Анне. Урок QA: DOM-проверки были зелёными при БИТОЙ картинке (эмулятор игнорирует
Content-Type при загрузке — чинится PATCH-ом метаданных); проверка усилена до
`naturalWidth > 0`, скриншот смотрится глазами.
**Проверка:** `node tools/verify-batch2.mjs` — фото отрисовано, лайтбокс в профиле и
связях, кнопки не перепутаны.

## Симптом
Тап по фотографии человека (профиль, связи) ничего не делает. Должен открывать фото
во весь экран.

## Причина
`Avatar.svelte` — статический кружок без обработчика. Дополнительная беда: на стенде
фотографий «не бывает» (`avatarUrl` возвращает `null` на localhost — эмулятор Storage
не поднят), поэтому поведение фото на стенде вообще непроверяемо.

## План фикса
1. `Avatar.svelte`: фото есть → кружок становится кнопкой (zoom-in), тап открывает
   лайтбокс-оверлей (фикс-слой, фото по центру, тап/Esc закрывают). Буква (нет фото) —
   не кнопка, разворачивать нечего.
2. «Связи»: вынести аватар из кнопки-шапки карточки (вложенные интерактивы невалидны);
   тап по имени раскрывает карточку, тап по фото — лайтбокс.
3. **Стенд должен уметь фото** (канон «стенд похож на бой»): поднять эмулятор Storage
   (порт 9199) в `npm run stand`, сид кладёт аватар dev-пользователю и одному соседу,
   `avatarUrl` больше не отрезает стенд, `storage()` на стенде идёт в эмулятор.
   `storage.rules` заводится для эмулятора (в бой НЕ выпускается — это bugs/04).

## План проверки
Стенд, Playwright: профиль → тап по фото → лайтбокс виден, фото в нём то же; Esc/тап —
закрылся; связи → тап по фото соседа — лайтбокс, тап по имени — раскрытие карточки
(не перепутаны). Скриншоты.
