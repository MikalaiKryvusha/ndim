# Bug 26 — Синяя браузерная обводка при тапах по кнопкам

**Статус:** ✅ DONE (2026-07-24, в бою) · ранее: починен, проверен QA-прогоном на стенде (2026-07-16), ждёт выката ·
**Источник:** волна 09 (п. 8)

**Фикс:** глобально в `+layout.svelte`: `-webkit-tap-highlight-color: transparent`,
`:focus:not(:focus-visible) { outline: none }`; клавиатурный фокус видим — свой стиль в
тонах темы (`:focus-visible`).
**Проверка:** `node tools/verify-batch3.mjs` — после клика outline `none`, после Tab —
видимый.

## Симптом (слово владельца)
«При тапах по кнопкам появляется дефолтная синяя обводка браузера — убрать по всему
приложению. В оригинальном NDim не было».

## Причина
Дефолтные стили браузера: `outline` на `:focus` после тапа + мобильный tap-highlight.
Собственных стилей фокуса у продукта нет, поэтому виден системный.

## План фикса — не ломая доступность
1. Глобально в `+layout.svelte`: `-webkit-tap-highlight-color: transparent` и
   `:focus:not(:focus-visible) { outline: none }` — тап больше не рисует обводку.
2. **`:focus-visible` оставить видимым** (аккуратный стиль в тонах темы, не системный
   синий): клавиатурная навигация не должна ослепнуть — это не «украшение», а доступность.
3. Пройтись по всем интерактивам (кнопки, ссылки, звёзды, табы) в Chrome тапами и Tab-ом.

## План проверки
Playwright: клик по кнопкам — computed `outline-style` в состоянии фокуса после клика =
none; навигация Tab-ом — фокус ВИДЕН. Скриншоты до/после тапа.
