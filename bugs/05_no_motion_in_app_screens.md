# Bug 05 — Внутренним экранам не хватает анимаций: всё резкое, топорное

**Статус:** 🔧 фикс сделан и выкачен (f657668 + c4acf46), ждёт полной регрессии по `plans/05` ·
**Заведён:** 2026-07-12 (боль владельца, записана агентом по его прямой просьбе)
**Важность:** высокая — это лицо продукта. Владелец сравнивает внутренности с лендингом не в их пользу.

## Симптом (слово владельца, 2026-07-12, дословно)

> «По всему приложению не хватает анимаций. Лендинг делал Fable 5 — получилось красиво.
> Внутренности приложения делал Opus 4.8 по планам, которые оставил Fable 5 — получилось
> топорно и с багами. Opus плохо справился с внутренностями приложения».

Это уже **второе** повторение той же боли: в претензиях к экрану «Измерения» (2026-07-12,
шапка `design/dims-mockups.html`) владелец писал «нет анимаций, всё резкое», и утверждённый
макет обещал «всё с анимацией, ничего резкого» — реализация выполнила обещание лишь точечно.

## Криминалистика — инвентаризация движения (2026-07-12)

Счёт CSS-переходов / keyframes / Svelte-переходов (`transition:`/`in:`/`out:`/`animate:`) по файлам:

| Файл | CSS transitions | @keyframes | Svelte-переходы |
|---|---|---|---|
| `+page.svelte` (лендинг, **Fable 5**) | 5 | 1 (pulse) | 0 |
| `ui/SimilarityDemo.svelte` (демо, **Fable 5**) | 6 | 3 | 0 |
| `dims/+page.svelte` | 6 | 2 | 0 |
| `profile/+page.svelte` | **0** | **0** | **0** |
| `relations/+page.svelte` | **0** | **0** | **0** |
| `space/+page.svelte` | **0** | **0** | **0** |
| `menu/+page.svelte` + все 8 страниц | **0** | **0** | **0** |
| `ui/AppBar`, `ui/BottomNav`, `ui/Avatar` | **0** | **0** | **0** |
| `ui/SideRail` | 1 | 0 | 0 |

Итог: живое движение есть только там, где работал Fable 5 (лендинг + демо). **Профиль, Связи,
Пространство, Меню — ноль анимаций**: ни перехода состояния, ни появления контента, ни hover.
«Измерения» лучше прочих (уезд карточки вправо, тост «Отменить», спиннер), но и там: карточки
ленты появляются рывком, соседи удалённой карточки прыгают вверх без FLIP, раскрытие описания
резкое, меню ⋮ возникает мгновенно, смена вкладок и поиск — рывком.

## Корневая причина

В проекте не существует **языка движения**: нет канона длительностей и кривых, нет привычки
использовать встроенные средства Svelte (`transition:fade/slide`, `animate:flip`), хотя стек даёт
их бесплатно (best practice вместо велосипеда — принцип `MASTER_PLAN`). Каждый внутренний экран
верстался «до работоспособности» — движение не считалось частью продукта. Механизм ошибки тот же,
что с жаргоном (EXP-0032): агент делает ФУНКЦИЮ, а не ПРОДУКТ.

## План фикса

1. **Канон движения** — короткий раздел в `AGENT_GUIDE.md` («Дизайн» → «Движение») + токены
   длительностей/кривых в `+layout.svelte`. Планка — лендинг: спокойно, 150–350 мс, ничего не
   прыгает. Уважать `prefers-reduced-motion` (лендинг уже так делает — наследуем).
2. **Правило:** всё, что появляется/исчезает/меняет состояние на экране, делает это с переходом;
   списки перестраиваются через `animate:flip`; раскрытия — `slide`.
3. Пройти экран за экраном: «Измерения» (вместе с bugs/06) → «Профиль» → «Связи» →
   «Пространство» → «Меню» и его страницы → `AppBar`/`SideRail`/`BottomNav` (переходы active).
4. Проверка: глазами по стенду + там, где движение несёт смысл (уезд карточки), — e2e на класс.
