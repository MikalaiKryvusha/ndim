# Bug 40 — Базовая ссылка открывает лендинг вместо приложения; нет загрузочного экрана 1.x

**Статус:** 🔴 открыт · **Источник:** владелец, чат 2026-07-17

## Симптом (слово владельца)
«Когда открываю по базовой ссылке приложение — открывает всегда лендинг и не переходит
внутрь на главную, несмотря на то, что я авторизован. В оригинальном NDim был загрузочный
экран, который не показывал приложение, пока оно определяло состояние пользователя, а
затем красиво открывало нужный экран, без дёрганых переходов».

## Что не так сейчас
1. **Редирект не срабатывает у владельца.** bugs/10 (пачка 1, 08.1) сделал редирект
   вошедшего с лендинга в `/profile` — но у владельца на проде он не происходит
   (или происходит не всегда). Воспроизвести и найти причину: `src/lib/data/session.ts`
   + onMount лендинга; возможные кандидаты — гонка инициализации Auth (redirect ждёт
   `onAuthStateChanged`, который на холодном старте может прийти поздно), кеш статики,
   ранний выход по ошибке.
2. **Канон 1.x — загрузочный экран.** Даже когда редирект работает, человек видит
   «дёрганый» переход: лендинг мелькает, потом прыжок. В 1.x был boot-экран: приложение
   не показывалось, пока не определено состояние пользователя, затем красиво открывался
   нужный экран.

## План фикса
1. Разведка (plans/06): снять boot-флоу живого 1.x (`ndim-old`: index.html/app.js —
   как выглядел загрузочный экран, что проверял) → `researches/12`. Каноничное кольцо
   «( )» уже есть — `Loading.svelte` (bugs/21).
2. Воспроизвести невыполнение редиректа: прод, авторизованная сессия, холодное открытие
   `https://ndimspace.app/` (обычное и в новой вкладке). Читать консоль.
3. Решение по канону 1.x: на базовой ссылке при НАЛИЧИИ признака сессии (быстрая
   эвристика — ключ Firebase Auth в IndexedDB/localStorage) показывать boot-экран
   (кольцо «( )»), дождаться `onAuthStateChanged`, затем плавно (канон `motion.ts`)
   открыть `/profile`; гостю/новому — лендинг как сейчас. Без мелькания лендинга.
   ⚠️ Пререндер и SEO лендинга не ломать: boot-слой — клиентский, поверх статики;
   счётчик `landing_view` продолжает считаться только для тех, кто ОСТАЁТСЯ на лендинге.

## План проверки
Стенд: вошедший открывает `/` → кольцо → плавно `/profile`, без кадра лендинга
(скриншоты по таймлайну/трейс Playwright); гость и разлогиненный видят лендинг;
`landing_view` не растёт у вошедших (юнит/e2e на funnel уже есть — сверить);
e2e лендинга (пререндер, темы, языки) зелёные. Прод: слово владельца после выката.
