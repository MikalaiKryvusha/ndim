# Bug 11 — Виджет «Добро пожаловать» показывался старым пользователям при каждом входе

**Статус:** ✅ DONE (2026-07-16, в бою) · **Источник:** волна 08 (п. 2), ручной тест владельца

## Симптом
При входе по ссылке из письма человек видел поздравление о регистрации («Профиль сохранён ·
Добро пожаловать в Пространство NDim · Аккаунт создан») — даже если он давний пользователь
и просто залогинился. Поздравление уместно только для того, кто вот-вот зарегистрировался.

## Воспроизведение
Существующим аккаунтом (например, человеком из 1.x) войти по почтовой ссылке → виджет.

## Причина
`finishEmailLink()` в `src/routes/profile/+page.svelte` ставил `signupStep = 'done'` при
ЛЮБОМ успешном `completeLoginLink()`: слой аккаунта не сообщал экрану, был ли это апгрейд
гостя/новый аккаунт или обычный вход. Заодно воронка `account_created` засчитывала входы.

## Фикс (сделан)
Флаг `created` в `UpgradeResult` (`src/lib/data/account.ts`): `linkWith*` (апгрейд гостя) →
`true`; `signInWith*` → `getAdditionalUserInfo().isNewUser`. Экран показывает «done»-карточку
и шлёт `account_created` только при `created: true`.

## Проверка (пройдена)
`node tools/verify-bugs08.mjs`: оба пути почтовой ссылки через oob-коды эмулятора Auth
(EXP-0045) — существующий входит БЕЗ поздравления, свежий гость получает «Профиль сохранён».
