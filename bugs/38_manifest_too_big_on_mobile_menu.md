# Bug 38 — Манифест в мобильном меню слишком развесист

**Статус:** 🔧 починен (2026-07-17), проверка — QA-прогон verify-batch4 + e2e ·
**Источник:** владелец, чат 2026-07-17

## Симптом (слово владельца)
«В меню сейчас очень сильно развесистый манифест — уберем его для мобильной версии: пусть
в мобильной идут в специальную кнопку полного манифеста. В десктопе пусть этот манифест
будет — в десктопе полно места, чтобы два виджета повесить рядышком — манифест и меню кнопок».

## Решение (задано владельцем — макеты не требуются)
- **Телефон:** вместо манифеста — кнопка-строка «📜 Манифест» (первая карточка меню),
  ведущая на новую полную страницу **`/menu/manifesto`** (пререндерена, как остальные
  страницы раздела).
- **Десктоп (от 1024px):** манифест — полный виджет РЯДОМ с кнопками меню; сетка `/menu`
  стала трёхколоночной: манифест | аккаунт·вид·поделиться | документы·проект·версии.
- Текст манифеста вынесен в общий модуль `src/lib/content/manifest.ts` (один источник для
  виджета и страницы); «Читать дальше» больше не нужен — на десктопе текст целиком.
- Пререндер `/menu` по-прежнему содержит полный текст (виджет скрыт CSS) — «лицо проекта»
  в сыром HTML охраняется e2e без изменений.

## Проверка
`node tools/verify-batch4.mjs` (секция bugs/38): телефон — виджет скрыт, кнопка видна и
ведёт на полный текст; десктоп — виджет виден рядом с колонками кнопок, кнопка скрыта.
`e2e/menu.spec.ts` обновлён (viewport-зависимая проверка + `/menu/manifesto` в списке
страниц). Скриншоты обеих раскладок — глазами.
