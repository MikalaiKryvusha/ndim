## Шаг 2 — Вычистить старые данные из публичного репозитория

Ты выбрал (интервью №001, В5): тот же репозиторий `MikalaiKryvusha/ndim`, историю сбросить до чистого
состояния. Локально это уже сделано: новая история не содержит ни пароля, ни почт, ни ключа подписи.

Но **force-push не удаляет старые объекты с GitHub сразу.** Старые коммиты остаются доступны по прямым
ссылкам (`github.com/.../commit/<hash>`) до тех пор, пока GitHub не проведёт сборку мусора, — а это
может не произойти никогда, если на них ссылаются форки, PR или кеш.

Выбери способ:

- **A) (рекомендую) Удалить репозиторий и создать заново с тем же именем.**
  Самый надёжный способ: объекты исчезают вместе с репозиторием. URL `github.com/MikalaiKryvusha/ndim`
  сохраняется. Теряются: звёзды, watchers, issues, форки (если они есть).
  ```bash
  gh repo view MikalaiKryvusha/ndim --json stargazerCount,forkCount,issues
  # если ничего ценного не теряется:
  gh repo delete MikalaiKryvusha/ndim
  gh repo create MikalaiKryvusha/ndim --public --source . --push
  ```

- **B) Force-push + запрос в поддержку GitHub.**
  Сохраняет звёзды и issues. Но требует написать в поддержку с просьбой удалить «dangling»-объекты, и
  до этого момента старые коммиты доступны по хешу.

- **C) Сделать репозиторий приватным.**
  Мгновенно закрывает доступ. Но противоречит духу `GOAL.md` — проект задуман открытым.

**Твой выбор:** A 

> ⚠️ Прежде чем выбирать A, посмотри, есть ли у репозитория форки: если кто-то форкнул его, данные
> живут и в форке, и удаление оригинала их не тронет.
